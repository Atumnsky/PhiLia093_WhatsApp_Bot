"""
PhiLia093 WhatsApp 机器人

================================================================================
概览
================================================================================
这是一个适用于 WhatsApp 群的友好 AI 聊天机器人，基于 whatsapp-web.js 库构建。名为 PhiLia093，该机器人结合了对话式 AI、网络搜索和表情贴纸反应，创造温暖、有趣的群聊体验。

机器人使用：

whatsapp-web.js – 与 WhatsApp Web 交互。

DeepSeek API – 进行自然语言理解和生成。

Tavily 搜索 API – 实现实时网络搜索。

本地贴纸文件 – 根据提示发送表情贴纸，增加互动趣味。

机器人设计为安全、尊重、友好，遵循严格的安全规则，避免冒犯性语言、不当学习或有害互动。

================================================================================
功能特性
================================================================================

对话式 AI – 在群聊中对以 # 开头的消息作出响应。机器人使用温暖、温和的个性，有时会在句末添加音符 (♪)。

网络搜索 – 使用 #search <查询词> 通过 Tavily 获取实时信息。机器人会将搜索结果融入回答，并在末尾提供来源链接。

时间查询自动搜索 – 如果消息包含时间相关关键词（如“几点”、“现在时间”）但未以 #search 开头，机器人会自动启用搜索以给出准确答案。

贴纸反应 – AI 可以通过在回复中包含 [STICKER: 情绪]（例如 [STICKER: happy]）来建议发送贴纸。机器人会在文本消息之后从对应情绪文件夹中随机选择一张贴纸发送。可用情绪取决于放置在 ./stickers 目录中的文件夹。

每用户对话记忆 – 机器人会为每个用户（在群聊中按参与者）保留最近约 1500 个 token 的对话上下文。

冷却机制 – 对同一用户的消息实施 5 秒冷却，防止刷屏。

创建者特殊待遇 – 当创建者（ID 195035186348201@lid）发言时，机器人会添加额外的系统提示，使其更加温暖和亲切。

安全与内容过滤 – 系统提示包含严格规则：禁止冒犯性语言、学习脏话、使用生造词、讨论敏感话题（如 LGBTQ+）。机器人会礼貌拒绝此类请求。

上线/下线通知 – 机器人启动时，会向配置的群组（若设置了 ONLINE_GROUP_ID）发送“Online!”消息；优雅关闭时（Ctrl+C）发送“Offline!”消息。

启动时列出群组 – 机器人会记录其所在的所有群组，方便确认 ID。

================================================================================
命令
================================================================================
所有命令必须在群聊中发送，并以 # 开头：

# <你的消息> – 与 PhiLia093 正常聊天。

#search <查询词> – 执行网络搜索并获取带有来源的答案。

（对于时间查询，即使不加 #search 也会自动触发搜索。）

机器人不会响应不以 # 开头的消息。

================================================================================
配置（环境变量）
================================================================================

ONLINE_GROUP_ID – 发送上线/下线通知的 WhatsApp 群组 ID。若未设置，则不发送通知。

DEEPSEEK_API_KEY – 你的 DeepSeek API 密钥（必需）。

TAVILY_API_KEY – 你的 Tavily 搜索 API 密钥（搜索功能必需）。

此外，机器人硬编码了：

CREATOR_ID – 创建者的 WhatsApp ID（195035186348201@lid），用于特殊待遇。可根据需要修改。

贴纸必须放在 ./stickers 文件夹中，子文件夹以情绪命名（例如 happy、sad、wave）。每个情绪文件夹内放置图片文件（.webp、.png、.gif、.jpg）。机器人请求该情绪时会随机选取一张。

================================================================================
代码结构
================================================================================

导入 – 所需库。

配置 – API 密钥、ID、常量。

客户端初始化 – 使用 LocalAuth 创建 WhatsApp 客户端，保持会话持久。

记忆与冷却 – memory 对象存储每个用户的对话历史；cooldown 记录时间戳。

系统提示 – SYSTEM_PROMPT 定义个性、安全规则和行为。

辅助函数：

trimMemory(messages, maxTokens) – 限制对话历史的大致 token 数量。

loadStickers() – 扫描 ./stickers 文件夹，构建情绪 → 文件路径的映射。

sendSticker(chatId, emotion) – 发送指定情绪的随机贴纸。

searchTavily(query, maxResults) – 调用 Tavily API 并返回结果。

事件处理：

client.on('qr') – 显示用于身份验证的二维码。

client.on('ready') – 记录群组并发送上线消息。

client.on('message') – 主消息处理器。

优雅关闭 – handleExit() 发送下线消息并销毁客户端。

客户端初始化 – 启动机器人。

================================================================================
扩展/修改机器人
================================================================================

更改个性或规则：编辑 SYSTEM_PROMPT，调整语气、安全规则或贴纸建议逻辑。

添加新命令：修改消息处理器，检测新前缀（如 #weather）并实现自定义逻辑。

搜索行为：更改 searchTavily() 中的 maxResults，或修改自动搜索关键词列表。

贴纸系统：在 ./stickers 中添加新情绪文件夹。确保情绪名称与 AI 可能输出的内容匹配（不区分大小写）。也可以修改 sendSticker 以使用不同文件类型或发送逻辑。

记忆管理：调整 trimMemory 的 token 限制或最大历史长度（目前为每个用户最多 30 条消息，经过修剪）。

冷却：在消息处理器中更改冷却时间（当前为 5000 毫秒）。

创建者 ID：根据需要更新 CREATOR_ID 为其他 WhatsApp ID。

时区：修改 currentTimeStr 生成中的 timeZone 以匹配你的本地时间。

错误处理：增强 try/catch 块以获得更稳健的恢复。

================================================================================
注意事项与限制
================================================================================

机器人仅在 群聊 中响应（消息来自 @g.us），忽略私聊。

机器人使用 whatsapp-web.js，需要有效的 WhatsApp Web 会话。首次运行时，需用 WhatsApp 手机应用扫描二维码。后续运行通过 LocalAuth 保存会话。

贴纸需为支持格式（.webp 最适合贴纸）。机器人通过 sendMediaAsSticker: true 将其作为贴纸发送。

AI 偶尔会建议贴纸；可通过调整系统提示控制频率。

机器人没有像 Discord 版本那样的持久化知识库；所有记忆在内存中，重启后丢失。

自动搜索功能基于简单关键词匹配，可能不够完美。

确保 Tavily API 密钥有足够配额；机器人会对每个 #search 和自动检测到的时间查询调用它。

DeepSeek API 可能有速率限制或费用；请监控使用情况。

机器人仅处理以 # 开头的消息，避免回复无关内容，减少群聊干扰。
